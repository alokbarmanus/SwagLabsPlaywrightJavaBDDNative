🔄 LATEST UPDATE - SCENARIO ITERATION FIX
═════════════════════════════════════════════════════════════════════════════

Issue Fixed:
  Scenario was running only once despite having 2+ JSON records

Root Cause:
  testDataList was stored as Hooks instance variable
  PicoContainer creates new Hooks instance per iteration
  Data was lost between iterations

Solution Applied:
  ✅ Moved testDataList to ScenarioContext (shared across iterations)
  ✅ Added: setTestDataList(), getTestDataList(), hasTestDataList() methods
  ✅ Updated Hooks to check if data already loaded before re-reading JSON
  ✅ Scenario now runs multiple times - once per JSON record

Result:
  ✅ With 2 JSON records: Scenario runs 2 times
  ✅ With N JSON records: Scenario runs N times
  ✅ Each iteration uses different data from JSON
  ✅ Proper data-driven execution working!

See: SCENARIO_ITERATION_FIX.txt for detailed explanation


🚨 CRITICAL SETUP REQUIREMENT - READ THIS FIRST!
═════════════════════════════════════════════════════════════════════════════

The implementation uses Constructor Dependency Injection for ScenarioContext.

REQUIRED: Add this dependency to pom.xml (ALREADY ADDED)

  <dependency>
      <groupId>io.cucumber</groupId>
      <artifactId>cucumber-picocontainer</artifactId>
      <version>${cucumber.version}</version>
  </dependency>

This enables Cucumber to:
  ✓ Inject ScenarioContext into Hooks constructors
  ✓ Inject ScenarioContext into Step Definitions constructors
  ✓ Use the same instance across Hooks and Steps

If you get this error without this dependency:
  CucumberException: class com.qa.hooks.Hooks does not have a public
  zero-argument constructor.

Solution: Ensure cucumber-picocontainer is in pom.xml (already done)
Then run: mvn clean install && mvn test


WHAT WAS IMPLEMENTED:
═════════════════════

Three new utility classes were created and one existing class was updated
to enable reading JSON test data and running scenarios multiple times.


1. NEW FILE: JsonDataReader.java
   Location: src/main/java/com/qa/utils/JsonDataReader.java
   
   Purpose:
   ├─ Reads JSON files from disk
   ├─ Parses JSON using Jackson library
   ├─ Converts to List<Map<String, Object>> for easy access
   └─ Supports nested objects and dot notation paths
   
   Key Methods:
   ├─ List<Map<String, Object>> getDataAsListOfMaps() → Get all records
   ├─ Map<String, Object> getDataAtIndex(int) → Get specific record
   ├─ Object getValue(Map, fieldName) → Get value with dot notation
   ├─ Object getNestedValue(Map, fieldPath) → Navigate nested objects
   ├─ int getTotalRecords() → Count records
   └─ void printAllData() → Debug output
   
   Example Usage:
       JsonDataReader reader = new JsonDataReader("path/to/file.json");
       List<Map<String, Object>> allData = reader.getDataAsListOfMaps();
       String city = (String) reader.getValue(allData.get(0), "address.city");


2. NEW FILE: ScenarioContext.java
   Location: src/main/java/com/qa/utils/ScenarioContext.java
   
   Purpose:
   ├─ Acts as shared data holder
   ├─ Injected into Hooks and Step Definitions
   ├─ Tracks iteration progress (current record, total records)
   └─ Enables communication between Hooks and Steps
   
   Key Methods:
   ├─ void setTestData(Map<String, Object>) → Store current data set
   ├─ Map<String, Object> getTestData() → Get current data set
   ├─ Object getTestDataValue(String fieldName) → Get specific value
   ├─ void setCurrentDataIndex(int) → Track which record
   ├─ int getCurrentDataIndex() → Get current position
   ├─ void setTotalDataRecords(int) → Store total count
   ├─ int getTotalDataRecords() → Get total
   ├─ boolean isLastRecord() → Check if last iteration
   └─ void printContextData() → Debug output
   
   Example Usage:
       public StepDef(ScenarioContext context) { this.context = context; }
       Map<String, Object> data = context.getTestData();
       String username = (String) data.get("username");


3. UPDATED FILE: Hooks.java
   Location: src/main/java/com/qa/hooks/Hooks.java
   
   Changes:
   ├─ Added constructor injection of ScenarioContext
   ├─ Added new @Before(order=1) readJsonDataFile(Scenario)

   │  └─ Triggers automatic scenario re-run

   Execution Order:
   @Before(order=0) → getProperty()

   [STEPS EXECUTE]

KEY CONCEPTS:
═════════════

1. DEPENDENCY INJECTION
   └─ Cucumber's PicoContainer automatically injects ScenarioContext
   └─ Same instance is injected into Hooks and Steps
   └─ No manual creation needed

2. SCENARIO RE-RUNNING
   └─ When @After sets new data and context updates
   └─ Cucumber framework automatically re-runs the scenario
   └─ @Before hooks execute again with fresh setup
   └─ Steps execute with new data

3. DATA PERSISTENCE
   └─ testDataList held in Hooks instance variable
   └─ ScenarioContext shared across Hooks and Steps
   └─ Data updated in @After for next iteration

4. NESTED DATA HANDLING
   └─ JSON nested objects become nested Maps
   └─ Cast to Map<String, Object> to access
   └─ Use dot notation through getTestDataValue()
   └─ Example: "address.city" traverses nested structure
