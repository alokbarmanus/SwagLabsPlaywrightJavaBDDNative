4. INJECT ScenarioContext IN STEP DEFINITIONS
   ──────────────────────────────────────────
   
   public class LoginPageStepDefs {
       private ScenarioContext scenarioContext;
       
       public LoginPageStepDefs(ScenarioContext scenarioContext) {
           this.scenarioContext = scenarioContext;
       }
       
       @Given("user is on the login page")
       public void user_is_on_the_login_page() {
           loginPageObjects = new LoginPageObjects(DriverFactory.getPage());
           loginPageObjects.launchApplication();
       }
       
       @When("user enters username.*from data file")
       public void user_enters_username_and_from_data_file(...) {
           // Get CURRENT record from context
           Map<String, Object> testData = scenarioContext.getTestData();
           
           String username = (String) testData.get("username");
           String password = (String) testData.get("password");
           
           loginPageObjects.enterUsername(username);
           loginPageObjects.enterPassword(password);
       }
       
       @Then("user should see the dashboard header text")
       public void user_should_see_the_dashboard_header_text(String expectedHeader) {
           loginPageObjects = new LoginPageObjects(DriverFactory.getPage());
           assertThat(loginPageObjects.getDashboardPageHeaderLocator())
               .hasText(expectedHeader);
           
           // 🔑 THE MAGIC: Check if more records to process
           if (scenarioContext.hasMoreRecords()) {
               scenarioContext.moveToNextRecord();
               executeScenarioForCurrentRecord(expectedHeader);
           }
       }
       
       // Helper method that recursively executes steps for remaining records
       private void executeScenarioForCurrentRecord(String expectedHeader) {
           loginPageObjects.logout();  // Reset state
           
           // Re-execute steps for next record
           user_enters_username_and_from_data_file("username", "password");
           user_clicks_login_button();
           
           loginPageObjects = new LoginPageObjects(DriverFactory.getPage());
           assertThat(loginPageObjects.getDashboardPageHeaderLocator())
               .hasText(expectedHeader);
           
           // Check again recursively
           if (scenarioContext.hasMoreRecords()) {
               scenarioContext.moveToNextRecord();
               executeScenarioForCurrentRecord(expectedHeader);
           }
       }
   }

HOW IT WORKS:
═════════════

1st Iteration (automatic):
  Given user is on login page
    → Launch app
  When user enters username/password
    → Gets testData[0] from context (standard_user)
  And user clicks login
    → Logs in
  Then user should see dashboard
    → Verifies ✓
    → Checks: More records? YES!
    → Moves to testData[1] (visual_user)
    → Calls executeScenarioForCurrentRecord()

2nd Iteration (inside executeScenarioForCurrentRecord):
  → Logout from current user
  → Enter new username/password (visual_user)
    → Gets testData[1] from context
  → Click login
  → Verify dashboard ✓
    → Checks: More records? NO!
    → Returns from recursion

KEY LEARNING POINTS:
════════════════════

1. Cucumber runs scenarios once - so we handle looping INSIDE the scenario
2. ScenarioContext stores ALL data, not just current record
3. Each iteration gets fresh data from context via moveToNextRecord()
4. Logout between iterations to reset application state
5. Recursive method calls handle "how many iterations" automatically
6. Single scenario execution = cleaner test reports + efficient browser reuse

IMPORTANT NOTES:
════════════════

• @dataFile tag is REQUIRED for parameterized execution
• JSON data MUST be an array [ ]
• {env} in path is replaced with environment value from application.properties
• ScenarioContext is injected automatically by Cucumber
• For nested data use: testData.get("parent") then ((Map) parent).get("child")
• Or use: scenarioContext.getTestDataValue("parent.child")


WHAT HAPPENS WHEN TEST RUNS:
════════════════════════════

Record 1 (standard_user):
  ✓ @Before reads JSON
  ✓ Sets Record 1 in context
  ✓ Scenario runs with Record 1 data
  ✓ @After closes browser, sets Record 2

Record 2 (visual_user):
  ✓ Scenario runs again with Record 2 data
  ✓ @After closes browser, all done


ACCESSING DATA IN STEPS:
════════════════════════

Simple fields:
    String username = (String) scenarioContext.getTestData().get("username");

Nested objects:
    Map<String, Object> address = (Map) scenarioContext.getTestData().get("address");
    String city = (String) address.get("city");

Using dot notation:
    String city = (String) scenarioContext.getTestDataValue("address.city");

Get iteration info:
    int current = scenarioContext.getCurrentDataIndex();    // 0-based
    int total = scenarioContext.getTotalDataRecords();
    boolean isLast = scenarioContext.isLastRecord();
