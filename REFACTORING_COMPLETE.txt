🎯 WHAT WAS FIXED
═════════════════════════════════════════════════════════════════════════════

Problem:
  Scenario was running only ONCE despite having 2 records in JSON file
  Expected behavior: Scenario runs once per JSON record
  Actual behavior: Only first record was being tested

Why This Happened:
  testDataList was stored as an instance variable in Hooks class
  Cucumber's PicoContainer creates a NEW instance of Hooks per iteration
  When the second iteration started, the first Hooks instance was discarded
  Second iteration's new Hooks instance didn't have access to testDataList
  JSON was being re-read and index was reset to 0
  Scenario mistakenly processed Record 1 again instead of Record 2


✅ THE FIX
═════════════════════════════════════════════════════════════════════════════

Solution: Move testDataList from Hooks to ScenarioContext

Why This Works:
  ScenarioContext is injected via constructor
  Same instance persists across PicoContainer lifecycle
  Data stored in context is accessible to new Hooks instances
  Allows proper sequential processing of all JSON records

Changes Applied:

1. ScenarioContext.java (Enhanced)
   ├─ Added: private List<Map<String, Object>> testDataList;
   ├─ Added: public void setTestDataList(List<Map<String, Object>> dataList)
   ├─ Added: public List<Map<String, Object>> getTestDataList()
   ├─ Added: public boolean hasTestDataList()
   └─ Now stores ALL records from JSON array

2. Hooks.java (Refactored)
   ├─ Removed: private JsonDataReader jsonDataReader;
   ├─ Removed: private List<Map<String, Object>> testDataList;
   ├─ Removed: private int currentDataIndex;
   ├─ Updated readJsonDataFile():
   │  └─ Check: if (scenarioContext.hasTestDataList()) return;
   │  └─ Only reads JSON on FIRST iteration
   │  └─ Stores in context, not local variable
   └─ Updated quitBrowser():
      └─ Retrieves testDataList from context
      └─ Uses context for all index/data management

💡 KEY CONCEPTS
═════════════════════════════════════════════════════════════════════════════

1. PicoContainer Lifecycle
   └─ Creates new Hooks instance per scenario iteration
   └─ Injected ScenarioContext is shared/persistent
   └─ Data in context survives instance recreation

2. Instance Variables vs Shared Context
   └─ Instance var: Lost when instance is discarded
   └─ Context data: Persists across instances (same reference)

3. Constructor Injection Benefits
   └─ Injected objects have longer lifecycle than instance
   └─ Enables sharing data across multiple instances

4. Data-Driven Testing Pattern
   └─ Store shared data in injected context
   └─ Not in instance/local variables
   └─ Enables proper iteration through test data
